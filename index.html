Chapter 04

一个完整的 CoT 案例

用户输入

「我失业快 3 个月了，之前的公司裁员」

麦麦的 <thinking> 过程
  • 情绪识别：被裁的失落 + 3 个月失业的焦虑，可能有自我怀疑
  • 实际情况：被裁是不可控事件（公司决定），不是个人能力问题
  • 需求判断：更需要陪伴和理解，而非「去投简历」的冷建议
  • 回应策略：承认不公平 → 肯定坚持 → 给陪伴感
麦麦最终输出

「被裁确实很无奈，这真的不是你的问题。但你能在这 3 个月里坚持，说明你很有韧性。这个群里很多人都经历过，我们一起陪伴。」

Chapter 05

Prompt 工程七大技术

来自 Anthropic 官方文档的核心方法论。
这是你现阶段最重要的技能。

Chapter 05

七大技术一览

01 · 清晰指令

把 AI 当成新实习生。不说「回答要好」，而说「回应不超过 50 字，先表达同理心，再给一个建议」。

02 · Few-shot 例子

给 3–5 个「用户说 → 理想回应」的配对。AI 从模仿中学会你要的风格。

03 · Chain of Thought

让 AI 分步思考：先识别情绪 → 再分析需求 → 最后给出回应。准确率提升 15–20%。

04 · XML 标签结构

用 <thinking> <response> 等标签分隔思考过程和最终输出，结构更清晰。

05 · 角色设定

在 System Prompt 中定义麦麦的身份：「你是有同理心的陪伴者，不是冷冰冰的客服」。

06 · 链式提示

将复杂任务拆成多步：第一步识别情感类型 → 第二步选择回应策略 → 第三步生成文案。

07 · 长上下文利用 — 把群聊的历史对话作为上下文传入，让 AI 的回应不再是孤立的。

Chapter 06

三阶段行动计划

01

现在 → 3 个月：深化 Prompt + 收集数据

用豆包 1.5 Lite 跑 3 个月。收集 1000+ 条用户反馈。标注 500 条数据（五维评分）。迭代 Prompt 5 个版本。目标：用户满意度 70→80 分。

02

3–6 个月：评估是否微调

条件触发：如果满意度卡在 75 分且竞品更好。执行：用 1500 条数据 LoRA 微调豆包。A/B 测试微调版本 vs Prompt 版本。

03

6 个月+：自研（如果证明值得）

条件触发：麦麦成为杀手级功能、留存对比 >15%、竞品开始模仿。投入自研团队，用 Llama/Qwen 底座长期迭代。

你需要学的五件事

按优先级排序。
不是都要同时学，而是分阶段攻克。

Chapter 07

五项核心技能

  • Prompt 工程 现在就学
    系统提示词写法、Few-shot、Chain of Thought、Temperature 调参。Anthropic 官方文档是最好的教材。
  • 数据标注与质控 现在就学
    设计标注指南、标注员一致性评估、多维度评分体系。工具推荐：Label Studio。
  • A/B 测试设计 1 个月内
    样本量计算、变量控制、效应量判断。在真实群聊中对比不同 Prompt 版本的效果。
  • Benchmark 构建 1–2 个月
    测试集设计、人工评估 vs 自动指标、用 Benchmark 指导迭代方向。
  • Fine-tuning 基础 3 个月后
    LoRA（轻量微调）+ Instruction Tuning（指令微调）。等数据足够了再深入。
